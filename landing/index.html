讓知識比對話活得更久

MemTrace 是一個給團隊與 AI 代理人共用的開放知識平台。知識不必塞進龐大的文件——它拆成一顆顆聚焦的記憶節點,以帶型別的邊互相連結,共同長成一張活的知識圖譜。

Core Concepts

五個核心概念,一張活的圖

價值不在單一節點,而在它們形成的網路。新成員或 AI 代理人可以從任一節點進入,沿著邊走到所有相關的一切——不需要原作者在旁邊帶。

記憶節點 Memory Node

一個結構化、雙語(zh-TW / en)的知識單元,帶內容型別、可見性、標籤與走訪計數。一顆節點,一個想法。

知識圖譜 Knowledge Graph

節點以帶型別的邊連結:depends_on、extends、related_to、contradicts。同時被走訪的邊會變強。

衰退 Decay

邊的權重隨時間衰減;有人共同走訪時再加回 +0.10–0.30。沒人走的連結,最終會自然消散。

w(t) = w₀ × 0.5 ^ (days / half_life)

走訪追蹤 Traversal

每顆節點、每條邊都記錄被走訪幾次、被多少不同的行動者(人或 AI)走過。路徑會自我強化。

知識庫 Knowledge Base

多個工作空間,四種分享層級(公開 / 條件式公開 / 受限 / 私有)與邀請連結。節點可跨知識庫複製攜帶。

Knowledge Inheritance

為知識傳承而設計

它對人對人、人對 AI、AI 對 AI 的知識分享一樣有效。每個貢獻者都寫進同一張圖。

捕捉

把一個聚焦的想法寫成一顆節點——手動建立,或上傳文件讓 AI 提出候選節點、由你審核後再入庫。

連結

用帶型別的邊把節點接起來。有用的連結會因為被反覆走訪而變強,沒人走的會逐漸衰退消失。

傳承

新人或 AI 代理人從任一節點切入,沿邊探索到相關的一切——知識不再隨對話或離職一起消失。

Feature Overview

一個平台,涵蓋整條知識工作流

節點編輯

純文字或 Markdown、雙語標題、內容型別、標籤、可見性。

圖譜與路徑

型別邊、權重衰退、共走加成、路徑評分(1–5)、走訪計數。

AI 抽取

上傳文件 → AI 提出候選節點 → 使用者審核後才入庫。

自帶金鑰

OpenAI / Anthropic 使用者自帶 API Key,伺服器不留存。

來源與出處

節點記錄由誰、何時、以什麼工具或模型建立;上傳文件保留為參考來源。

驗證與登入

Email 驗證註冊、Magic Link 登入。

外部 API

REST API 搭配範圍化金鑰:kb:read / kb:write / node:traverse / node:rate。

引導上手

網頁精靈與互動式 memtrace init CLI 流程。

Ecosystem Integration

接進你現有的 AI 工作流

MemTrace 原生整合更廣的 AI 生態,讓你的知識圖譜可以直接用在標準 LLM 應用裡。

MODEL CONTEXT PROTOCOL

原生 MCP 伺服器

讓 Claude Desktop、Cursor、Antigravity 等工具直接搜尋、讀取、建立與更新圖譜節點——不必翻原始文件。支援 SSE 與 Streamable HTTP。

# Streamable HTTP(Cursor / Antigravity)
URL: https://<host>:8000/mcp
Auth: Bearer mt_<your_api_key>
OPENAI-COMPATIBLE API

OpenAI 相容端點

以 /v1/chat/completions 對選定工作空間做混合語意檢索(hybrid RAG),自動注入相關節點並回傳來源引用,支援 SSE 串流。

from openai import OpenAI
client = OpenAI(
  base_url="http://localhost:8000/v1",
  api_key="mt_xxx",
)
PYTHON SDK

官方 Python 客戶端

以程式化方式管理工作空間、建立/搜尋節點、進行 chat 與檢索。

from memtrace import MemTraceClient
c = MemTraceClient(api_key="mt_xxx")
c.search_nodes(query="how to config auth")
LANGCHAIN · LLAMAINDEX

框架 Retriever 整合

LCEL 相容的 MemTraceRetriever,以及 LlamaIndex 的 MemTraceVectorStore,幾行就接進既有 pipeline。

from langchain_memtrace import MemTraceRetriever
retriever = MemTraceRetriever(workspace_id="ws_abc")
chain = retriever | llm
Quick Start

幾分鐘就跑起來

PostgreSQL 17 + pgvector 由 docker compose 自動起,前後端各一行指令。完整步驟見 docs/DEVELOPMENT.md。

前往 GitHub 閱讀規格
bash — memtrace
# 1. 取得原始碼並安裝
git clone https://github.com/wei-nan/MemTrace.git
cd MemTrace && cp .env.example .env
npm install

# 2. 啟動資料庫(PostgreSQL 17 + pgvector)
docker compose up -d

# 3. 啟動 API
cd packages/api
python -m uvicorn main:app --reload

# 4. 啟動前端(另開終端)
cd packages/ui && npm run dev
