Core Concepts

Five concepts, one living graph

The value is not in any single node, but in the network they form. A new teammate or an AI agent can enter at any node and follow the edges to everything related — without the original author guiding them.

Memory Node

A structured, bilingual (zh-TW / en) unit of knowledge with a content type, visibility, tags, and a traversal counter. One node, one idea.

Knowledge Graph

Nodes connect through typed edges: depends_on, extends, related_to, contradicts. Co-accessed edges grow stronger.

Decay

Edge weight decays over time; co-access boosts it back by +0.10–0.30. Connections nobody follows eventually dissolve on their own.

w(t) = w₀ × 0.5 ^ (days / half_life)

Traversal Tracking

Every node and edge records how many times it has been visited, and by how many distinct actors (humans or agents). Useful paths reinforce themselves.

Knowledge Base

Multiple workspaces with four sharing levels (public / conditional_public / restricted / private) and invite links. Nodes are portable across knowledge bases.

Knowledge Inheritance

Designed for knowledge inheritance

It works equally well for human-to-human, human-to-AI, and AI-to-AI sharing. Every contributor writes into the same graph.

Capture

Write one focused idea as a node — create it manually, or upload a document and let AI propose candidate nodes for you to review before committing.

Connect

Link nodes with typed edges. Useful connections strengthen as they are traversed; unused ones decay and eventually fade away.

Inherit

A newcomer or AI agent enters at any node and explores outward along the edges — knowledge no longer leaves with the conversation or the person.

Feature Overview

One platform across the whole knowledge workflow

Node editing

Plain text or Markdown, bilingual titles, content type, tags, visibility.

Graph & paths

Typed edges, weight decay, co-access boost, path ratings (1–5), traversal counts.

AI extraction

Upload a document → AI proposes candidate nodes → you review before commit.

Bring your own key

User-supplied OpenAI / Anthropic API key; never stored server-side.

Provenance

Each node records who created it, when, and with which tool or model; uploaded documents are kept as reference sources.

Auth

Email registration with verification, Google OAuth 2.0.

External API

REST API with scoped keys: kb:read / kb:write / node:traverse / node:rate.

Onboarding

Guided web wizard and the interactive memtrace init CLI flow.

Ecosystem Integration

Plug into your existing AI workflow

MemTrace ships native integrations with the broader AI ecosystem, so your knowledge graph drops straight into standard LLM apps.

MODEL CONTEXT PROTOCOL

Native MCP server

Let Claude Desktop, Cursor, Antigravity and others search, read, create, and update graph nodes directly — no raw spec docs. Supports SSE and Streamable HTTP.

# Streamable HTTP (Cursor / Antigravity)
URL: https://<host>:8000/mcp
Auth: Bearer mt_<your_api_key>
OPENAI-COMPATIBLE API

OpenAI-compatible endpoint

/v1/chat/completions runs hybrid semantic search (RAG) over the selected workspace, injecting relevant nodes and returning source citations, with SSE streaming.

from openai import OpenAI
client = OpenAI(
  base_url="http://localhost:8000/v1",
  api_key="mt_xxx",
)
PYTHON SDK

Official Python client

Manage workspaces, create/search nodes, and run chat & retrieval programmatically.

from memtrace import MemTraceClient
c = MemTraceClient(api_key="mt_xxx")
c.search_nodes(query="how to config auth")
LANGCHAIN · LLAMAINDEX

Framework retrievers

An LCEL-compatible MemTraceRetriever and a LlamaIndex MemTraceVectorStore — a few lines into your existing pipeline.

from langchain_memtrace import MemTraceRetriever
retriever = MemTraceRetriever(workspace_id="ws_abc")
chain = retriever | llm
Quick Start

Up and running in minutes

PostgreSQL 17 + pgvector come up via docker compose; one line each for API and UI. Full guide in docs/DEVELOPMENT.md.

Open on GitHub Read the spec
bash — memtrace
# 1. Clone and install
git clone https://github.com/wei-nan/MemTrace.git
cd MemTrace && cp .env.example .env
npm install

# 2. Start the database (PostgreSQL 17 + pgvector)
docker compose up -d

# 3. Start the API
cd packages/api
python -m uvicorn main:app --reload

# 4. Start the UI (new terminal)
cd packages/ui && npm run dev
